# Расширение: WordFormatterExtension

## Описание

Блок предназначен для автоматической очистки и унификации форматирования документов Microsoft Word. Позволяет удалять лишние пробелы, пустые абзацы и табуляции, применять единый шрифт ко всему документу, автоматически определять заголовки по нумерации и присваивать им стили, а также форматировать таблицы. Дополнительно поддерживает замену текстовых плейсхолдеров вида `{{key}}` на значения из словаря.

**Блок полезен для:**
- Подготовки шаблонных документов к печати или публикации
- Приведения документов из разных источников к единому корпоративному стандарту
- Автоматической генерации отчётов с подстановкой переменных

## Требования к окружению

- Операционная система: Windows 10/11
- Установлен Microsoft Word (2016 или новее)
- Puzzle RPA Studio 3.0.2 или новее

## Установка расширения

1. Скачайте архив `WordFormatterExtension.zip` и распакуйте его в любую папку.

2. Откройте Puzzle RPA Studio → **Настройки** → **Расширения** → **Добавление**.

3. Нажмите **Выбрать** и укажите папку с распакованным расширением.

4. Выберите категорию «Интеграция приложений / Word» или создайте новую.

5. Перезапустите Puzzle RPA Studio.

После перезапуска блок появится в тулбоксе в указанной категории.

## Параметры блока

| Параметр | Тип | Обязательный | Значение по умолчанию | Описание |
|----------|-----|--------------|----------------------|----------|
| source_path | string | Да | — | Полный путь к исходному `.docx` файлу |
| output_path | string | Нет | — | Путь для сохранения результата (если не указан, файл будет перезаписан) |
| font_name | string | Нет | Times New Roman | Основной шрифт документа |
| font_size | int | Нет | 14 | Размер шрифта в пунктах |
| placeholders | dict/string | Нет | — | Словарь для замены плейсхолдеров, формат: `{"{{ключ}}": "значение"}` |

## Пример использования в Puzzle RPA

1. Перетащите блок на рабочую область.

2. Заполните обязательные поля:
   - `source_path`: `C:\test\document.docx`
   - (опционально) `output_path`: `C:\test\document_formatted.docx`

3. При необходимости укажите `font_name` и `font_size`:
   - `font_name`: `Times New Roman`
   - `font_size`: `14`

4. Если в документе есть плейсхолдеры, например `{{user_name}}` и `{{current_date}}`, передайте в `placeholders` JSON-строку или готовый словарь:

```json
{"{{user_name}}": "Баранова Софья", "{{current_date}}": "23.05.2026"}
```

5. Запустите выполнение (F5).

## Демонстрационные файлы

- `Input.docx` – тестовый документ до обработки (с плейсхолдерами, разным форматированием, пустыми строками)
- `Output.docx` – результат работы блока на этом документе (эталон)
- `Input1.docx` – результат работы блока на этом документе (эталон)
- Ссылка на видео: https://drive.google.com/file/d/1I0ZHUBRIv2d09EuqLKx_oiZ_XzLNDCAS/view?usp=sharing

## Известные ограничения

- Работает только с форматом `.docx` (не поддерживается старый формат `.doc`)
- Требуется установленный Microsoft Word (любая версия от 2016 и новее)
- Плейсхолдеры должны быть написаны точно в том же регистре и с теми же пробелами, что и в документе (например, `{{user_name}}` ≠ `{{User_Name}}`)
- Заголовки 1 уровня распознаются по шаблонам: `1. Текст`, `ВВЕДЕНИЕ`, `ЗАКЛЮЧЕНИЕ`, `ОСНОВНАЯ ЧАСТЬ`

## Обработка ошибок

Блок содержит встроенную валидацию параметров и выводит понятные сообщения об ошибках в Puzzle RPA Studio.

### Возможные ошибки

| Ситуация | Сообщение |
|----------|------------|
| Не указан путь к исходному файлу | `Не указан путь к исходному Word файлу` |
| Исходный файл не найден | `Исходный файл не найден: C:\path\file.docx` |
| Указан неподдерживаемый формат файла | `Поддерживаются только Word файлы .docx` |
| Размер шрифта не является числом | `Размер шрифта должен быть числом. Получено: 'abc'` |
| Размер шрифта меньше или равен 0 | `Размер шрифта должен быть больше 0` |
| Размер шрифта вне допустимого диапазона | `Размер шрифта должен быть в диапазоне от 6 до 72` |
| Указан несуществующий шрифт | `Шрифт 'Лорпа' не найден в системе Windows` |
| Word не смог сохранить файл | `Word не смог сохранить файл: C:\path\file.docx. Проверьте доступ к папке и открыт ли файл.` |

### Особенности поведения

- Если параметр `output_path` не указан, документ будет перезаписан.
- При возникновении ошибки Microsoft Word автоматически закрывается.
- Блок не изменяет структуру документа, а только форматирует существующее содержимое.
- При указании несуществующего шрифта выполнение будет остановлено до начала форматирования документа.

### Рекомендации

- Используйте только установленные в Windows шрифты.
- Не открывайте документ в Word во время работы блока.
- Для корректной работы рекомендуется использовать локальные пути (`C:\...`), а не сетевые диски.
- Перед массовой обработкой документов рекомендуется протестировать блок на копии файла.
